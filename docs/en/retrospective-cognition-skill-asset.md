# Retrospective Cognition Skill Asset

Status: source of truth
Language: English
Last updated: 2026-03-26

## Purpose

This skill is used after a meaningful work round has completed and the result
should be sedimented into durable cognition assets instead of remaining only in
chat context.

## Trigger Conditions

Use this skill when:

- a debugging or rollout round finishes
- a repeated blocker has been exposed
- a stable user correction should persist
- a task reaches a checkpoint that should survive beyond the current session

## Core Actions

1. Reload the relevant source-of-truth documents.
2. Separate verified state from inference.
3. Identify what should persist and what should not.
4. Write the smallest durable artifact that preserves the lesson.
5. Promote only stable lessons into canonical layers.

## Guardrails

- Do not mix internal private detail into public-safe artifacts.
- Do not promote a one-off observation into a stable rule without evidence.
- Do not confuse narrative recap with durable cognition.
