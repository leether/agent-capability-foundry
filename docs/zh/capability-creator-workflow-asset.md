## Workflow

1. 命名意图。
2. 声明 capability 边界。
3. 把工作切分为阶段。
4. 给每个阶段附上约束。
5. 在阶段之间附上门控。
6. 评估当前门是否已通过。
7. 如果范围包含公开导出，检查 release topology。
8. 编写英文真源文档。
9. 派生中文 companion 文档。
10. 在继续前冻结当前阶段。

## 输出预期

输出应包含：

- 意图
- 阶段列表
- 阶段约束
- 门控标准
- 发布拓扑判断
- 当前阶段判断
- 下一步判断
