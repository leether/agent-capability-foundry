# Capability Creator Validation Example

Status: source of truth
Language: English
Last updated: 2026-03-26

## Validation Intent

The output should be considered valid only if it includes:

- an explicit intent
- explicit phases
- phase-specific constraints
- gate criteria
- a current phase judgment
- an English-source / Chinese-derived rule

## Failure Example

The output fails validation if it:

- gives a generic plan with no gates
- skips phase judgment
- mixes implementation promises into an earlier design phase
- treats localization as optional after the fact
