# Attribution

`agent-capability-foundry` is an original repository with its own governing
methodology, capability-first object model, curation rules, and runtime-aware
installation surface.

It is also explicitly informed by prior public agent-skill repositories and
their installation patterns.

## Referenced Upstream

- Original referenced source:
  - `charon-fan/agent-playbook`
- Current GitHub canonical location:
  - `zhaono1/agent-playbook`
- Referenced upstream asset:
  - `skills/self-improving-agent`
- Upstream license:
  - `MIT`

## Relationship

This repository does not present the upstream work as its own.

The current repository adapts and extends ideas from that upstream, especially
around self-improvement / self-evolution workflows, while changing the repo
model substantially.

## Major Downstream Changes

- capability-first packaging instead of skill-only top-level packaging
- explicit methodology constraints and curation gates
- agent-native, capability-oriented repository positioning
- English-source / Chinese-derived documentation policy
- runtime-aware adapter structure for multiple runtimes
- validation and safety assets as first-class capability content

## Attribution Guidance

Where a capability or asset is directly adapted from an upstream public source,
the repository should say so explicitly using wording such as:

- `Inspired by`
- `Adapted from`
- `Contains modified material from`

The wording should match the degree of derivation rather than flattening all
cases into a vague acknowledgement.
