# Retrospective Cognition

Use this capability after a meaningful work round has completed and the result
should be sedimented into durable cognition assets instead of remaining only in
chat context.

## Trigger Conditions

- a debugging or rollout round finishes
- a repeated blocker is exposed
- a stable user correction should persist
- a checkpoint should survive beyond the current session

## Core Actions

1. Reload the relevant source-of-truth documents.
2. Separate verified state from inference.
3. Identify what should persist and what should not.
4. Write the smallest durable artifact that preserves the lesson.
5. Promote only stable lessons into canonical layers.
