# Capability Creator

Use this capability when a new capability should be created through a governed
workflow instead of unstructured drafting.

## Trigger Conditions

- a new capability idea appears
- the capability needs explicit phases and gates
- bilingual source/derived docs are required
- runtime-aware packaging must be planned early

## Core Actions

1. Restate the intent.
2. Define the boundary.
3. Partition the work into phases.
4. Define constraints for each phase.
5. Define cross-phase gates.
6. Judge whether the next phase may be entered.
7. Write English source first, then derive Chinese.
